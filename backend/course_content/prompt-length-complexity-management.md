# Prompt Length & Complexity Management / 提示长度与复杂度管理

## Overview / 概述

Learn techniques for managing prompt length and complexity for optimal performance. / 学习管理提示长度和复杂度的技术，优化性能。

## Key Knowledge Points / 核心知识点

### 1. Length Management / 长度管理

**English:** Length management involves controlling prompt token count, balancing information completeness with context window limits.

**中文:** 长度管理涉及控制提示的token数量，平衡信息完整性和上下文窗口限制。

**Key Concepts / 核心概念:**
- Token limits / Token限制
- Information density / 信息密度
- Context window / 上下文窗口

**Example / 示例:**
```
Instead of long explanations, use concise bullet points and clear instructions.
→ Optimized prompt length

```

---

### 2. Complexity Management / 复杂度管理

**English:** Complexity management reduces prompt complexity through task decomposition, simplified instructions, and template usage.

**中文:** 复杂度管理通过分解任务、简化指令、使用模板等方式降低提示的复杂度。

**Key Concepts / 核心概念:**
- Task decomposition / 任务分解
- Simplified instructions / 简化指令
- Template reuse / 模板复用

**Example / 示例:**
```
Break complex task into: 1) Analyze, 2) Process, 3) Format
→ Simplified multi-step approach

```
