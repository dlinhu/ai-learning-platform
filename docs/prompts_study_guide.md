# 提示词工程知识点串讲大纲

> 本大纲涵盖提示词模块的核心知识点，每个知识点配有简单示例，帮助学员快速理解。

---

## 第一部分：基础篇 - 提示词的原子与分子

### 1. 原子 Prompt (Atoms)

**概念**：提示词的最小组成单元，包含三个核心要素。

**三要素**：
- **TASK（任务）**：要做什么
- **CONSTRAINTS（约束）**：怎么做
- **OUTPUT（输出）**：输出什么格式

**示例**：
```
TASK: 将以下英文翻译成中文
CONSTRAINTS: 保持专业术语不变，使用正式书面语
OUTPUT: 仅输出翻译结果，不要解释

Input: "Machine Learning is a subset of Artificial Intelligence."
```

---

### 2. 最小 Prompt (Minimal Prompt)

**概念**：最简洁有效的提示模板，只包含必要信息。

**示例**：
```
翻译成中文：Hello World
```

**对比**：
```
❌ 冗余版：请你帮我翻译一下这句话，把它翻译成中文，谢谢
✅ 最小版：翻译成中文：Hello World
```

---

### 3. 零样本 (Zero-shot)

**概念**：不给任何示例，直接让模型完成任务。

**示例**：
```
将以下句子分类为正面、负面或中性：

"这家餐厅的服务态度很好，但菜品一般。"
```

---

### 4. 少样本 (Few-shot)

**概念**：提供几个示例，让模型学习模式。

**示例**：
```
将以下句子分类为正面、负面或中性：

示例1: "今天天气真好！" → 正面
示例2: "这电影太无聊了。" → 负面
示例3: "我在吃午饭。" → 中性

输入: "这家餐厅的服务态度很好，但菜品一般。"
输出:
```

---

### 5. 分子 Context (Molecules)

**概念**：将多个原子组合成更复杂的上下文结构。

**示例**：
```
【角色】你是一位资深Python开发工程师
【任务】审查以下代码并指出问题
【约束】
- 关注代码规范
- 检查潜在bug
- 提供改进建议
【输出格式】
问题列表 + 改进后的代码

代码：
def calc(x,y):
    return x+y
```

## 第二部分：进阶篇 - 推理与控制

### 6. 思维链 (Chain of Thought, CoT)

**概念**：让模型展示推理过程，逐步思考。

**示例**：
```
问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？

请一步步思考：
```

**模型输出**：
```
步骤1：小明原来有5个苹果
步骤2：给了小红2个，剩下 5-2=3个
步骤3：又买了3个，现在有 3+3=6个
答案：6个苹果
```

---

### 7. 角色扮演 (Role/Persona)

**概念**：给模型设定特定身份，增强回答的专业性和针对性。

**示例**：
```
你是一位有20年经验的儿科医生。一位新手妈妈问：
"宝宝8个月大，最近总是哭闹，不愿意吃奶，怎么办？"

请给出专业建议。
```

---

### 8. 约束生成 (Constrained Generation)

**概念**：限制输出的格式、长度、风格等。

**示例**：
```
用一句话（不超过20字）描述春天的特点。

约束：
- 必须包含"花"字
- 使用比喻手法
```

---

### 9. 自洽性 (Self-Consistency)

**概念**：让模型多次推理，选择最一致的答案。

**示例**：
```
问题：一个袋子里有3个红球和2个蓝球，随机取出2个，都是红球的概率是多少？

请用三种不同的方法计算，然后比较结果。
```

---

### 10. 扩展上下文 (Expand Context)

**概念**：通过添加背景信息增强模型理解。

**示例**：
```
【背景】
- 产品：智能手表
- 目标用户：25-35岁都市白领
- 核心卖点：健康监测、运动追踪、消息提醒

【任务】
写一段30秒的广告文案。
```

## 第三部分：复杂任务篇 - 分解与串联

### 11. 指令工程 (Instruction Engineering)

**概念**：编写清晰、无歧义的指令。

**好的指令示例**：
```
请将以下JSON数据转换为Markdown表格：
- 第一列为字段名
- 第二列为数据类型
- 第三列为说明
- 按字母顺序排列字段

数据：{"name": "张三", "age": 25, "city": "北京"}
```

**坏的指令示例**：
```
把这个变成表格
```

---

### 12. 任务分解 (Task Decomposition)

**概念**：将复杂任务拆分为多个子任务。

**示例**：
```
复杂任务：写一篇关于人工智能的文章

分解为：
1. 子任务1：列出文章大纲（5个要点）
2. 子任务2：为每个要点写一段（每段100字）
3. 子任务3：写开头和结尾
4. 子任务4：整合并润色
```

---

### 13. 提示链 (Prompt Chaining)

**概念**：将多个提示串联，前一个输出作为后一个输入。

**示例**：
```
【链路1】生成主题
输入：科技领域
输出：人工智能在教育中的应用

【链路2】生成大纲
输入：人工智能在教育中的应用
输出：1.现状 2.优势 3.挑战 4.展望

【链路3】扩展内容
输入：大纲
输出：完整文章
```

## 第四部分：上下文管理篇

### 14. 记忆单元 (Cells of Context)

**概念**：组织和管理对话中的上下文信息。

**示例**：
```
【用户画像】
- 姓名：张三
- 职业：产品经理
- 目标：学习数据分析

【对话历史】
- Q1: 什么是SQL？ A: SQL是结构化查询语言...
- Q2: 如何学习？ A: 建议从基础语法开始...

【当前问题】
你推荐什么学习资源？
```

---

### 15. 控制循环 (Control Loops)

**概念**：通过迭代优化输出质量。

**示例**：
```
任务：写一首关于春天的诗

循环1：生成初稿
循环2：检查押韵 → 修改
循环3：检查意境 → 润色
循环4：最终版本
```

## 第五部分：工程化篇

### 17. 提示编程 (Prompt Programming)

**概念**：用类代码的方式组织提示。

**示例**：
```
function analyze_sentiment(text) {
  prompt = `
    分析以下文本的情感：
    输入：${text}
    输出格式：{sentiment: "positive/negative/neutral", confidence: 0-1}
  `
  return call_llm(prompt)
}

result = analyze_sentiment("今天心情不错")
```

---

### 18. Prompt 程序 (Prompt Programs)

**概念**：组合多个提示形成程序流程。

**示例**：
```
# Prompt程序：文章生成器

step1 = prompt("生成文章主题", domain="科技")
step2 = prompt("生成大纲", topic=step1.output)
step3 = prompt("扩展内容", outline=step2.output)
step4 = prompt("润色文章", content=step3.output)

final_output = step4.output
```

---

### 19. 优化技巧 (Optimization Techniques)

**常用优化方法**：

| 技巧 | 说明 | 示例 |
|------|------|------|
| 简化语言 | 去除冗余 | "请帮我..." → "..." |
| 明确格式 | 指定输出格式 | "用JSON格式输出" |
| 添加示例 | Few-shot | 提供输入输出示例 |
| 分步引导 | Step by step | "请分三步完成" |
| 角色设定 | Persona | "你是专家..." |

**优化示例**：
```
❌ 优化前：
帮我写个东西，关于那个什么的，你知道的

✅ 优化后：
写一篇500字的科普文章，主题是"量子计算的基本原理"，目标读者是高中生，使用通俗易懂的语言。
```

### 20. 效果评估 (Evaluating Effectiveness)

**评估维度**：
- **准确性**：输出是否正确
- **一致性**：多次输出是否稳定
- **相关性**：是否回答了问题
- **可读性**：输出是否清晰

**评估示例**：
```
测试用例：
输入："北京的天气怎么样？"
期望输出：包含温度、天气状况的回答

评估：
- 准确性：是否提供了真实天气数据
- 一致性：多次询问是否得到相似结果
- 相关性：是否回答了天气问题
- 可读性：表达是否清晰
```


## 实践建议

1. **循序渐进**：从原子开始，逐步掌握复杂技巧
2. **多做实验**：每个技巧都动手尝试
3. **对比效果**：同一任务用不同方法对比结果
4. **记录总结**：建立自己的提示词库
5. **持续优化**：根据效果不断改进提示

## 常见问题

**Q: 零样本和少样本如何选择？**
A: 简单任务用零样本，复杂或需要特定格式时用少样本

**Q: 什么时候用思维链？**
A: 数学推理、逻辑分析、复杂决策时使用

**Q: 如何提高输出一致性？**
A: 使用明确的约束条件、固定格式、提供示例

**Q: 提示词越长越好吗？**
A: 不是，要在清晰和简洁之间平衡，避免冗余信息
